🎯 Project’s Central Objective

Develop an integrated model for analysis and optimization of agricultural commodity portfolios, combining:

  1. Advanced Forecasting (GARCH/MSGARCH + ML/DL)
  2. Multi-Objective Optimization (NSGA-II, Differential Evolution)
  3. Reinforcement Learning (PPO, adaptive strategies)

Goal: Create tools for risk management and dynamic allocation in commodity markets, with potential for patent and scientific publications.

Interactive Mindmap

Detailed Project Structure

🎯 Three Methodological Pillars

PILLAR 1: Forecasting

Objectives: - Forecast expected returns of agricultural commodities - Estimate conditional volatilities and risk regimes - Capture complex temporal dependencies

Main Methods: - GARCH/MSGARCH: Volatility modeling with regime changes (Markov-Switching) - Machine Learning: LSTM, MLP, XGBoost, LightGBM - Deep Learning: Deep neural networks for non-linear patterns

Tools: - R: rugarch, MSGARCH, forecast - Python: torch, keras, xgboost, statsmodels

PILLAR 2: Multi-Objective Optimization

Objectives: - Determine efficient portfolio compositions - Balance return, risk, and diversification - Generate Pareto frontier with non-dominated solutions

Main Methods: - NSGA-II: Non-dominated Sorting Genetic Algorithm II - Differential Evolution: Global evolutionary optimization - SPEA2: Strength Pareto Evolutionary Algorithm 2

Tools: - R: DEoptim, mco, nsga2R - Python: pymoo, DEAP, scipy.optimize

PILLAR 3: Reinforcement Learning

Objectives: - Develop dynamic and adaptive trading strategies - Simulate tactical allocation decisions - Implement reallocation policies under uncertainty

Main Methods: - PPO: Proximal Policy Optimization - Gym Environment: Market simulation for training - Strategies: Momentum, dynamic hedging, trigger-based rebalancing

Tools: - Python: stable-baselines3, gym, ray[rllib] - R: Integration via reticulate

📅 Official PAIC 2025/26 Timeline

Stage 1: Setup and Governance (Aug-Sep/2025)

🔧 Main Activities
  • PAIC 2025/26 Opening: August 13, 2025
  • Commitment Term Signing
  • Scope review and research questions definition
  • Computational environment setup (R/Python + Git/GitHub)
  • Data source mapping: YahooQuery, Quandl, FAO, B3, CME
  • Success criteria and risk management definition

Deliverables: - Detailed work plan - Structured Git repository - Data sources documentation

Stage 2: Data Collection and Curation (Sep-Oct/2025)

📊 Main Activities
  • Historical commodity series collection (corn, soybeans, meal, soybean oil)
  • Data cleaning and normalization (frequencies, outliers, contract adjustments)
  • Financial feature engineering: log-return, volatility, drawdown, technical indicators
  • Exploratory analysis and descriptive statistics

Deliverables: - Curated and documented dataset - Complete exploratory data analysis (EDA) - Automated collection scripts

Stage 3: Volatility and Return Modeling (Oct-Nov/2025)

🤖 Main Activities
  • GARCH/MSGARCH model estimation (asymmetric distributions, regimes)
  • ML/DL prototype development: LSTM, MLP for forecasting
  • Performance comparison between econometric and ML models
  • 📝 Partial Report 1: November 24, 2025
  • 🎤 13th SPPAIC: October 24, 2025

Deliverables: - Calibrated forecasting models - Partial Report 1 (methodology and preliminary results) - Presentation at 13th PAIC Symposium

Stage 4: Optimization and Frontiers (Nov/2025-Jan/2026)

🎯 Main Activities
  • NSGA-II and Differential Evolution implementation
  • Pareto frontier generation (risk-return-diversification trade-offs)
  • Sensitivity and robustness analysis of solutions
  • Comparison with benchmarks (buy-and-hold, risk parity, mean-variance)

Deliverables: - Functional multi-objective optimization algorithms - Efficient portfolios on Pareto frontier - Alternative scenario analyses

Stage 5: RL and Backtesting (Jan-Mar/2026)

🚀 Main Activities
  • PPO agent training for dynamic strategies
  • Backtesting with temporal validation (walk-forward)
  • Comparison with market benchmarks
  • 📝 Partial Report 2: March 1, 2026
  • 🎓 Qualification Seminar: March 15, 2026

Deliverables: - Trained and validated RL agent - Partial Report 2 (complete results) - Qualification presentation

Stage 6: Consolidation and Dissemination (Mar-Jul/2026)

📄 Main Activities
  • Scientific paper writing (April-June/2026)
  • 📬 Article submission: July 6, 2026
  • Peer review and corrections
  • Poster/presentation preparation
  • 🎤 14th SPPAIC: August 10, 2026

Deliverables: - Submitted scientific article - Academic poster for SPPAIC - Complementary didactic material - Final project documentation

### 🎯 Three Methodological Pillars

<div class="team-grid">

<div class="project-info">
#### PILLAR 1: Forecasting

**Objectives:**
- Forecast expected returns of agricultural commodities
- Estimate conditional volatilities and risk regimes
- Capture complex temporal dependencies

**Main Methods:**
- **GARCH/MSGARCH**: Volatility modeling with regime changes (Markov-Switching)
- **Machine Learning**: LSTM, MLP, XGBoost, LightGBM
- **Deep Learning**: Deep neural networks for non-linear patterns

**Tools:**
- R: `rugarch`, `MSGARCH`, `forecast`
- Python: `torch`, `keras`, `xgboost`, `statsmodels`
</div>

<div class="project-info">
#### PILLAR 2: Multi-Objective Optimization

**Objectives:**
- Determine efficient portfolio compositions
- Balance return, risk, and diversification
- Generate Pareto frontier with non-dominated solutions

**Main Methods:**
- **NSGA-II**: Non-dominated Sorting Genetic Algorithm II
- **Differential Evolution**: Global evolutionary optimization
- **SPEA2**: Strength Pareto Evolutionary Algorithm 2

**Tools:**
- R: `DEoptim`, `mco`, `nsga2R`
- Python: `pymoo`, `DEAP`, `scipy.optimize`
</div>

<div class="project-info">
#### PILLAR 3: Reinforcement Learning

**Objectives:**
- Develop dynamic and adaptive trading strategies
- Simulate tactical allocation decisions
- Implement reallocation policies under uncertainty

**Main Methods:**
- **PPO**: Proximal Policy Optimization
- **Gym Environment**: Market simulation for training
- **Strategies**: Momentum, dynamic hedging, trigger-based rebalancing

**Tools:**
- Python: `stable-baselines3`, `gym`, `ray[rllib]`
- R: Integration via `reticulate`
</div>

</div>
